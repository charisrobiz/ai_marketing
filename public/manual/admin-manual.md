## 1. 시스템 개요

### AutoGrowth Engine이란?

AutoGrowth Engine은 AI 직원 8명이 협업하여 마케팅을 자동화하는 플랫폼입니다.
제품 정보만 입력하면, 30일 마케팅 플랜 수립부터 광고 소재 제작, AI 심사위원 투표, 성과 분석까지 자동으로 진행됩니다.

### AI 직원 구성

| 이름 | 역할 | 담당 업무 |
|------|------|----------|
| 하나 | 본부장 | 전체 전략 총괄, 소재 검토, 팀 조율 |
| 민서 | 마케팅 전략가 | 30일 플랜 수립, 니치밴딩 전략 |
| 지우 | SEO 카피라이터 | 롱테일 키워드 분석, 카피라이팅 |
| 태양 | 퍼포먼스 마케터 | Meta/Google 광고 세팅, CPA 최적화 |
| 유나 | 크리에이티브 디렉터 | 비주얼 컨셉, 이미지 감수 |
| 도하 | 모션 디자이너 | 숏폼 영상 제작, UGC 챌린지 템플릿 |
| 시원 | 개발자 | 파이프라인 구축, 기술 지원 |
| 은지 | 데이터 엔지니어 | 투표 집계, 성과 분석, CPA 벤치마크 |

### 기술 스택

- **프론트엔드**: Next.js 16 + React 19 + Tailwind CSS
- **데이터베이스**: Supabase (PostgreSQL)
- **파일 스토리지**: Supabase Storage
- **인증**: Supabase Auth
- **배포**: Vercel
- **AI LLM**: OpenAI GPT-4o-mini / Claude Sonnet 4 / Gemini 2.0 Flash
- **이미지 생성**: Gemini Nano Banana 2
- **동영상 생성**: Runway Gen-4 Turbo

## 2. 로그인 및 인증

### 로그인

1. 사이트 접속 시 로그인 화면이 표시됩니다.
2. 관리자 이메일과 비밀번호를 입력합니다.
3. "ID 저장" 체크 → 다음 방문 시 이메일이 자동 입력됩니다.
4. "자동 로그인" 체크 → 브라우저를 닫았다 열어도 로그인 상태가 유지됩니다.

### 로그아웃

- 우측 상단 로그아웃 아이콘(→)을 클릭합니다.
- 자동 로그인이 꺼져 있으면 브라우저를 닫으면 자동으로 로그아웃됩니다.

### 계정 관리

- 관리자 계정은 Supabase 대시보드에서 직접 관리합니다.
- Authentication → Users 에서 계정 추가/삭제가 가능합니다.
- 회원가입 기능은 비활성화되어 있으므로, 등록된 계정만 로그인 가능합니다.

## 4. 캠페인 생성

### 캠페인 유형

캠페인 성격에 따라 4가지 유형 중 선택합니다. AI가 유형에 맞는 전략과 플랜을 자동으로 최적화합니다.

| 유형 | 기간 | 사용 사례 | AI 전략 특징 |
|------|------|----------|-------------|
| ⚡ **긴급/이벤트** | 1~3일 | 플래시 세일, 한정 이벤트, 긴급 프로모션 | FOMO 극대화, 카운트다운, 즉각 전환 유도 |
| 🚀 **단기 캠페인** | 1~2주 | 신제품 출시, 시즌 프로모션 | 빠른 A/B 테스트 → 위너 소재 집중 투입 |
| 📈 **표준 캠페인** | 30일 | 브랜드 성장, 유저 확보 | 니치밴딩 4주 (발견→진입→장악→확장) |
| 🏗️ **장기 캠페인** | 2~3개월 | 브랜드 빌딩, 시장 점유 | 월별 피봇, 브랜드+퍼포먼스 균형 |

#### 유형별 AI 전략 차이

- **긴급/이벤트**: 시간 제한 강조, 즉각적 행동 유도. 느긋한 전략 불가 - 모든 액션이 즉시 효과를 냅니다.
- **단기**: Week 1에서 위너 소재를 2~3일 내에 찾아, Week 2에 집중 투입합니다.
- **표준**: 니치밴딩 프레임워크로 Week 1(시장 검증) → Week 2(오가닉 성장) → Week 3(스케일링) → Week 4(리텐션) 순서로 진행합니다.
- **장기**: Month 1(시장 진입) → Month 2(성장 가속) → Month 3(브랜드 확립). 매월 데이터 기반으로 전략을 피봇할 수 있습니다.

### 생성 플로우 (5단계)

#### Step 1: 카테고리 선택
모바일 앱, 웹 서비스, 실물 상품, 보험/금융, 교육, 식음료, 패션, 기타 중 선택

#### Step 2: 기본 정보
- **캠페인 유형** (필수) - 위 4가지 유형 중 선택
- **제품/서비스 이름** (필수)
- **간단한 설명**
- **타겟 고객**
- **핵심 차별점**

#### Step 3: AI 심층 질문
카테고리별로 최적화된 5가지 심층 질문에 답변합니다.
이 정보는 AI가 더 정확한 마케팅 전략을 수립하는 데 활용됩니다.

#### Step 4: 미디어 업로드 (선택)
CEO가 참고 자료를 업로드합니다:
- **앱 스크린샷** → AI 이미지 생성 시 스타일 참고
- **제품 동영상** → Runway 동영상 생성의 소스로 활용
- **참고 이미지** → 광고 이미지 스타일 반영
- **설명 문서** → LLM 카피 생성 시 컨텍스트로 활용

각 파일에는 **설명**과 **AI 활용 용도**를 설정할 수 있습니다.

> 이 단계는 건너뛸 수 있습니다. AI가 텍스트 정보만으로도 소재를 생성합니다.

#### Step 5: 확인 + 미디어 생성 옵션
- 입력 정보를 최종 확인합니다.
- **AI 이미지 생성** 체크: Gemini로 각 소재별 마케팅 이미지 생성 (~$0.04/장)
- **AI 동영상 생성** 체크: Runway로 숏폼 영상 생성 (~$0.25/5초)
- **Figma 배너 합성** 체크: Figma 템플릿 기반 광고 배너 자동 생성
- API 키가 미등록이면 체크 시 경고 메시지가 표시됩니다.

## 6. AI 엔진 실행

### 실행 방법
캠페인 상세 페이지에서 **"AI 엔진 시작"** 버튼을 클릭합니다.

### 실행 과정 (자동)

1. **킥오프 미팅** - 본부장 하나가 캠페인 유형에 맞는 전략 브리핑, 팀원 역할 분배
2. **Phase 1: 플랜 생성** - 민서가 캠페인 유형(긴급/단기/표준/장기)에 맞는 마케팅 플랜 수립
3. **Phase 2: 소재 생성** - 지우(카피) + 유나(비주얼) 협업으로 크리에이티브 제작
4. **Phase 2.5: 미디어 생성** (옵션)
   - 이미지 생성 체크 시: 유나가 Gemini로 각 소재별 이미지 생성
   - 동영상 생성 체크 시: 도하가 Runway로 상위 3개 소재 숏폼 영상 생성
5. **Phase 2.7: Figma 배너 합성** (옵션)
   - Figma 배너 합성 체크 시: Figma 템플릿을 읽어 플랫폼별 광고 배너 자동 합성
6. **Phase 3: AI 투표** - 은지가 100인 심사위원단 투표 집계
7. **마무리** - 1위 소재 선정, 광고 세팅 준비 완료

### CEO 업로드 미디어 활용

- 카피 생성 시: 미디어 설명이 프롬프트에 자동 포함
- 이미지 생성 시: 참고 이미지의 스타일이 반영
- 동영상 생성 시: CEO 소스 영상이 Runway 입력으로 우선 사용

### 실시간 모니터링
캠페인 상세 → **팀 미팅** 에서 AI 직원들의 실시간 대화를 확인할 수 있습니다.

## 7. 소재 승인 프로세스

### 2단계 승인

1. **본부장(하나) 검토**
   - 캠페인 상세 → **소재 승인** 에서 "본부장 검토 시작" 클릭
   - 각 소재를 브랜드 일관성, 타겟 적합성, 비용 효율 기준으로 평가
   - 결과: 승인(7점+) / 수정 요청(4-6점) / 반려(3점-)

2. **CEO 최종 승인**
   - CEO 승인 레벨이 "CEO 최종 승인"이면 승인 대시보드에서 확인
   - 개별 승인/반려 또는 "전체 승인" 가능
   - 코멘트를 달아 피드백 전달 가능

### 자동 승인 정책
- Week 2 이후 소재는 본부장 승인만으로 자동 집행 (자동 정책 적용)
- CAC 30% 이상 급등 시 CEO 검토 알림 발송

## 8. 주간 리뷰 및 리포트

### 주간 리뷰 미팅
캠페인 상세 → 각 주차의 **"리뷰 미팅 시작"** 버튼 클릭

- AI 팀이 해당 주의 성과를 분석하고 다음 주 전략을 조정
- 데이터 보고 → 전략 토론 → 업무 조정 → 다음 주 계획

### 주간 리포트
캠페인 상세 → **주간 리포트** 에서 확인

- 주차별 설치 수, CAC, 오가닉/페이드 비율
- 주요 하이라이트 및 이슈
- 다음 주 액션 아이템

### 실시간 지표 (외부 API 연동 시)
- Meta Ads: 노출, 클릭, 지출, CTR, CPC
- Google Ads: 노출, 클릭, 비용, 전환
- Firebase: DAU, 신규 유저, 세션, 리텐션
- AppsFlyer: 설치, 오가닉/페이드 비율

### 연동된 서비스 (데이터 조회)

| 플랫폼 | API 비용 | 실제 서비스 비용 |

### Figma 연동 (배너 자동 합성)

## 11. Figma 배너 합성

### 개요

CEO가 Figma에 만든 광고 템플릿에 AI가 생성한 카피와 이미지를 자동으로 합성하여 **바로 집행 가능한 광고 배너**를 만듭니다.

### 동작 방식

```
CEO가 Figma에 광고 템플릿 제작 (플레이스홀더 레이어 포함)
    ↓
캠페인 생성 시 Figma 파일 URL 입력 + "Figma 배너 합성" 체크
    ↓
AI 엔진 실행 시:
  1. Figma REST API로 템플릿 구조 읽기
  2. 플랫폼별 프레임 자동 인식 (Instagram, TikTok, YouTube 등)
  3. 각 프레임에서 플레이스홀더 위치/크기 추출
  4. Gemini AI가 템플릿 + AI 카피 + AI 이미지를 합성
  5. 플랫폼별 최종 배너 PNG 내보내기 → Supabase Storage 저장
    ↓
바로 집행 가능한 광고 배너 완성
```

### Figma 템플릿 만드는 방법

#### 1. 프레임 만들기
플랫폼별 프레임을 만들고, 이름에 플랫폼명을 포함하세요:
- `Instagram_1080x1080` → 인스타그램 피드
- `TikTok_1080x1920` → 틱톡/릴스
- `YouTube_1280x720` → 유튜브 썸네일
- `Blog_Header` → 블로그 헤더

#### 2. 플레이스홀더 레이어 만들기
AI가 인식하는 레이어 이름 규칙:

| 레이어 이름 | 역할 | 예시 |
|------------|------|------|
| `hooking_text` | 후킹 문구 (메인 타이틀) | "3초 만에 완성!" |
| `copy_text` | 본문 카피 | 상세 설명 텍스트 |
| `sub_text` | 보조 텍스트 | 부가 설명 |
| `cta_text` | CTA 버튼 텍스트 | "지금 시작하기" |
| `product_image` | 제품 이미지 영역 | AI 생성 이미지 배치 |
| `background` | 배경 이미지 영역 | 배경 이미지 배치 |
| `logo` | 로고 영역 | 변경하지 않고 유지 |

#### 3. 디자인 팁
- 플레이스홀더 위치와 크기를 정확히 잡아두면 AI가 해당 위치에 콘텐츠를 배치합니다.
- 브랜드 컬러, 폰트, 레이아웃 등 디자인 시스템을 템플릿에 적용하면 배너에 자동 반영됩니다.
- 하나의 Figma 파일에 여러 플랫폼 프레임을 만들면 한 번에 모든 플랫폼 배너가 생성됩니다.

### 필요한 API 키
- **Figma API Key**: 관리자 설정 → 디자인 섹션에서 등록
- **Gemini API Key**: 배너 합성에 Gemini 멀티모달 AI 사용 (AI LLM API에서 등록)

### 비용
- Figma API 호출: 무료
- Gemini 배너 합성: ~$0.04/배너
- 3개 소재 × 3개 플랫폼 = 9개 배너 → ~$0.36 (~520원)

## 13. FAQ

### Q. AI 엔진이 작동하지 않아요.
→ 관리자 설정에서 AI LLM API 키가 최소 1개 이상 등록되어 있는지 확인하세요.

### Q. 이미지/동영상이 생성되지 않아요.
→ 캠페인 생성 Step 5에서 해당 옵션을 체크했는지 확인하세요.
→ 관리자 설정에서 Gemini API 키(이미지) 또는 Runway API 키(동영상)가 등록되어 있는지 확인하세요.

### Q. 캠페인 중간에 제품 정보를 변경하면?
→ 캠페인 상세 → 수정 버튼으로 변경 가능합니다.
→ 본부장 하나가 자동으로 변경 영향도를 분석하고, 팀에게 수정 지시를 내립니다.

### Q. CEO 승인 없이 바로 집행하려면?
→ 관리자 설정 → CEO 승인 레벨을 "자동 승인"으로 변경하세요.

### Q. 캠페인 중간에 이미지/동영상을 추가하려면?
→ 캠페인 상세 → "미디어 관리" 에서 언제든 파일 업로드 및 텍스트 메모가 가능합니다.

### Q. 긴급 이벤트 캠페인은 어떻게 만드나요?
→ 캠페인 생성 Step 2에서 "⚡ 긴급/이벤트" 유형을 선택하세요.
→ AI가 3일 기한의 FOMO/카운트다운 중심 전략을 자동으로 수립합니다.

### Q. Figma 배너가 생성되지 않아요.
→ 관리자 설정에서 Figma API 키와 Gemini API 키가 모두 등록되어 있는지 확인하세요.
→ Figma 파일 URL이 올바른지, 템플릿에 최소 300x300 이상의 프레임이 있는지 확인하세요.
→ 프레임 안에 hooking_text, copy_text 등 이름의 레이어가 있으면 더 정확한 합성이 됩니다.

### Q. 데모 캠페인을 만들고 싶어요.
→ 홈 화면의 "데모 시뮬레이션" 버튼을 클릭하면 SNAPTALE 데모 캠페인이 자동 생성됩니다.

### Q. 데이터는 어디에 저장되나요?
→ 모든 데이터는 Supabase (PostgreSQL)에 저장됩니다.
→ 파일은 Supabase Storage에 저장됩니다.
→ 서버는 Vercel에서 운영됩니다.
